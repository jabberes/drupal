XMPP Roundup 14: software

Contenido archivado

El contenido de la web se encuentra archivado y no se podrá crear nuevo contenido. Más información.

Enviado por naw el 24 April, 2010 - 14:18.
Tagged:
  • Desarrollo
  • Diario Jabber
  • Lanzamientos

Bienvenidos al catorceavo Roundup de la comunidad XMPP community, ésta es la sección dedicada a software nuevo y actualizado.

Este Roundup ha sido escrito por Nÿco y Neustradamus, con la ayuda de Kevin, Johann, David, Will y Florian.

Clientes

Pigeon
Pigeon es un cliente de chat para XMPP y ICQ disponible para la plataforma Windows Mobile.

Cleartext ESM (Enterprise Social Messaging)
David Banes de Cleartext, lanzará el cliente de escritorio beta de Cleartext ESM, un cliente de microblogging y chat XMPP.

open-chats-xmpp 0.2 para Pidgin
Matej Cepl ha escrito open-chats-xmpp, unos scripts Perl simples que abren los marcadores marcados para autoconectar en Pidgin usando Pubsub para el almacenamiento de los marcadores.

Poezio
Poezio es un cliente XMPP/Jabber de estilo IRC: te permite conectar anónimamente a salas de chat (MUC) pero no tiene charla uno-a-uno incorporada.

Jabbin
Jabbin, el fork de Psi (y el único cliente que ha implementado el [protocolo] desaprobado TINS en el pasado), ha resucitado y está disponible en beta.

Oyo
Oyo es un cliente web de chat que se basa en la biblioteca Smack.

Pandion 2.6.90
Sebastiaan Deckers ha anunciado el lanzamiento de Pandion 2.6.90, con bastantes mejoras, incluyendo una arquitectura e interfaz rediseñadas, así como un cambio la la licencia GPLv3.

qutIM
qutIM es un "mensajero instantáneo multiprotocolo (ICQ, Jabber/GTalk/Ya.Online/LiveJournal.com, Mail.Ru, IRC) libre para sistemas Windows y Linux".

Beem for Android
Beem es un cliente XMPP para teléfonos y dispositivos Android, lanzado balo la licencia GPLv3.

OneTeam
OneTeam, un cliente XMPP de escritorio desarrollado por ProcessOne desde 2006, es ahora en alfa privada.

MCabber 0.10.0
MCabber es un completo cliente XMPP de consola para sistemas de tipo Unix.

OneTeam para GTalk y TextOne
ProcessOne ha lanzado dos nuevas aplicaciones para iPhone. La primera es un cliente de chat para GTalk con push, y la segunda es TextOne, una aplicación de mensajería que reemplaza la aplicación SMS, sin presencia, pero con chats y push.

Papaya 0.2
Papaya Project ha lanzado Papaya 0.2 (versión Mobile basada en Fennec).

Jappix
Jappix es un nuevo cliente XMPP para la web. Desarrollado por Vanaryon (Valérian Saliou), y publicado bajo la licencia AGPL en su versión 0.1, parece bonito y prometedor.

Servidores

Jingle Nodes para ejabberd
Johann Prieur ha contribuido un modulo de Jingle Nodes para ejabberd: mod_jinglenodes.

WebSockets for Prosody
Ali Sabil ha contribuido un module de WebSocket para Prosody: mod_websocket.

xmpp.js: Server-side XMPP en Javascript
Mathew Wild ha lanzado, como se anuncio en el FOSDEM, su biblioteca xmpp.js basada en Node.js.

Prosody 0.7.0 RC1
Prosody 0.7.0 RC1 está listo para testeo. Incluye un gran numero de mejoras.

Tigase XMPP server 5.0.0
Tigase XMPP server 5.0.0 está disponible para la descarga. El changelog es largo (hasta el final de la página), pero como resumen, esta versión añade Component Connections (XEP-0225), soporte de stringprep, correcciones en s2s, un nuevo framework de configuración del servidor, scripting disponible para todos los componentes del servidor.

Bibliotecas

StropheCappuccino
StropheCappuccino es una biblioteca JavaScript de bindings para el Cappuccino framework, que permite aplicaciones web en tiempo real.

Missus
Missus es una biblioteca PHP en temprano estado de desarrollo, lanzada bajo licencia LGPL.

Ahoy
Ahoy es una biblioteca Ruby para mensajería sin servidor usando Bonjour/DNSDS/mDNS and XMPP.

Los que no se ajustan a las categorías anteriores (¡pero molan!)

twitter-xmpp bot
Uriah Welcome ha desarrollado twitter-xmpp, un bot simple que hace de pasarela de Twitter a XMPP, con oauth.

PAD-XMPP: People Are Ducks
Tim Bielawa ha lanzado PAD-XMPP, un proyecto personal para aprender erlang y XMPP.

karaka: pasarela XMPP a Skype
Neil Stratford ha lanzado karaka, un transporte XMPP para Skype, usando la API de Skype, escrito en Python y publicado bajo la GPLv2.

Archipel: un organizador de virtualización basado en XMPP
Archipel, un software AGPL desarrollado por Antoine Mercadal, es un un organizador de virtualización basado en XMPP.

Spectrum 0.2: una pasarela/transporte XMPP
Spectrum, un software GNU GPL desarrollado por Jan Kaluža, es una pasarela/transporte XMPP para varias redes: AIM, Facebook, Gadu-Gadu, ICQ, IRC, MSN, SIMPLE, Twitter y XMPP. La segunda versión estable ha sido lanzada.

Edición compartida de HTML+CSS en tiempo real con Kompozer: sxEdit
Según Fabien Cazenave, quien dio una brillante charla en FOSDEM, KompoZer, el editor de HTML+CSS basado en Gecko recibirá características de edición compartida en tiempo real basadas en XMPP: sxEdit es el nombre del producto.

Pocho el robot
Mauro Pompilio ha lanzado Pocho, un bot en Ruby bot para salas de charla.

Bonita obtiene un conector XMPP
Bonita es un software BPM (Business Process Management) de código abierto. Ha recibido contribuciones para un conector XMPP.

Omnipresence
Grégoire Ménuel ha lanzado Omnipresence, un servicio de presencia para web, que permitirá a los usuarios mostrar su presencia en los sitios web.

OSAMI
El servicio de organización OSAMI ha añadido un bundle XMPP.

idzHXR
idzHXR es un plugin con una licencia MIT que permite Strophe funcionar sin ninguna limitación debido a las restricciones de las políticas crossdomain.

Ohm Studio
Ohm Studio es un secuenciador de audio colaborativo en tiempo real que permitirá a cualquier músico editar y mezclar música colaborativamente con cualquier otro músico en el mundo.

Presentación controlada remotamente
Christopher Zorn ha modificado s5 (herramienta de presentación basada en web) para controlar remotamente una presentación via XMPP. Usa la biblioteca Strophe.js library, el cliente Speeqe y todo funciona sobre XMPP (BOSH) y PubSub.

Aquí acaba este Roundup. Le animamos a hablar de él en cualquier lado y a contribuir a él.
