Nuevo Psi 0.9.2 - Ahora con transferencia de ficheros

Tagged:

Acaba de salir una nueva versión del popular cliente multiplataforma [Psi]. Después de casi 5 meses de espera, son bastantes los cambios en usabilidad que va a notar el usuario final. Entre otros:

  • Transferencia de ficheros (usando el nuevo estándar), y con gestor de transferencias
  • Autocompletado de apodos
  • Truco del día
  • Modo de chat compacto
  • Y los típicos arreglos...

Puedes descargarlo ya: Windows / MacOS X / Fuentes / Más descargas

Actualmente hay varias personas ocupadas actualizando las traduccionces españolas: MF el castellano, Necry el catalán y Non el gallego. Quien esté interesado en colaborar en estos u otros idiomas puede dirigirse al foro de Psi.

Lista completa de novedades (en inglés):

  - File transfer
  - Event queue is saved to disk at all times for fault protection
  - Enable/disable accounts
  - Groupchat nickname tab-completion
  - Groupchat trackbar to mark last read text when unfocused
  - *bold*, /italic/, _underline_
  - Fixed bug where the roster window could get lost on Windows
  - 'Compact' mode for chat window
  - Tip of the day
  - Hidden roster group
  - Ability to export history as readable text
  - Event queue sorting by type
  - Type-ahead search in the roster
  - External programs can now have parameters
  - Toolbar configuration is now accessed from the right-click menu
  - Show dates in local format
  - Use the gnome-open utility for opening URLs with GNOME2
  - Fixed "broken pipe" crash when receiving GnuPG errors

Transferencia de ficheros

Enviado por badlop el 12 June, 2004 - 15:41.

Según parece el antiguo sistema de transferencias era este:
JEP-0066: Out of Band Data.

El nuevo sistema de transferencia de ficheros se describe en
JEP-0096: File Transfer.

Dicha transferencia se negocia a través de los servidores Jabber (usando
JEP-0095: Stream Initiation), pero la transferencia en sí se puede realizar por dos vias:

  • existe la posibilidad de transferir por dentro de Jabber, en cuyo caso la transferencia irá del emisor a su servidor, de ahí al otro servidor, y de ahí al receptor. Esto no es deseable, claro, porque ralentiza ambos servidores y la propia transferencia. Descrito en
    JEP-0047: In-Band Bytestreams (IBB).
  • la posibilidad más deseable es no involucrar a los servidores, como está descrito JEP-0065: SOCKS5 Bytestreams. En este caso hay dos posibilidades:
    • conexión directa: el emisor y receptor de la transferencia establecen una conexión TCP directa para transferir el fichero.
    • conexión mediada: en la que emisor y receptor no se comunican directamente, sino a través de un Proxy SOCKS5. Supongo que esto será útil si el emisor o el receptor no quiere que la otra parte conozca su dirección IP.

Si no me he equivocado poniendo números y nombres, puedes transferir ficheros por Jabber si te puedes conectar a Jabber :). No obstante, para usar las transferencias directa o mediada hará falta tener algun puerto libre, claro. Lo que no he encontrado es una indicación del puerto por el que se escucha: el puerto de emisión y recepción lo especifican los clientes durante la negociación, según parece.

Pruebas con PSI

Enviado por Anónimo el 13 June, 2004 - 15:44.

Después de probar a transferir ficheros con el nuevo PSI, pongo aquí las conclusiones.

El nuevo Psi tiene, dentro de las Opciones, dos campos a rellenar para las transferencias: el puerto (8010 por defecto) y la IP externa.

- Si uno de los dos usuarios tiene el puerto 8010 accesible desde el exterior, la transferencia funciona en los dos sentidos.

- En el caso de que los dos usuarios tengan cerrado el puerto, la transferencia no funciona. Para arreglarlo es necesario que uno de ellos o bien permita las conexiones en el firewall o bien mapee el puerto en el router.

En este último caso, además, no basta con configurar el router, sino que además es necesario rellenar la casilla con la IP externa en las Opciones del programa, lo que puede resultar pesado. Sin embargo, he comprobado que en este campo funcionan los servidores DNS dinámicos como dyndns o no-ip, con lo que basta con escribir el nombre del dominio y es él el que se encarga de redireccionar a la IP actual.

Saludos :)
