Ha salido nueva versión de
Ejabberd,
el servidor Jabber libre más potente.

Descarga para Linux y *BSD o instalador para Windows.
Tutorial de instalación en Linux.

Lista de mejoras

  • Soporte de STARTTLS en conexiones cliente2servidor (ahora solo falta que las implemente servidor2servidor) (1)
  • Soporte de autentificación mediante script externo
  • Nuevo módulo que implementa JUD y vCard usando LDAP
  • Mejoras en la administración via web (crear/borrar usuarios, añadir y quitar contactos de su lista, ver mensajes diferidos)
  • Soporte para announce/all y announce/motd (enviar mensaje a todos los conectados; poner mensaje del dia) (2)
  • Soporte para captura de mensajes usando una herramienta externa, como Bandersnatch (ya están trabajando para que no haga falta herramienta externa)
  • Soporte para expiración de mensaje (JEP-0023)
  • Soporte para búsqueda en JUD con prefijo
  • Añadido límite en el número de resultados en una búsqueda por JUD
  • Soporte de HTTPS en la interfaz web (3)

Modificaciones en ejabberd.cfg

(1) Actualiza la sección listen:

{5222, ejabberd_c2s, [{access, c2s}, {shaper, c2s_shaper},
                      starttls, {certfile, "./ssl.pem"}]},
{5223, ejabberd_c2s, [{access, c2s},
                      tls, {certfile, "./ssl.pem"}]},

(2) Añade en la sección modules:

{mod_announce, [{access, announce}]},

(3) Reemplaza en la sección listen

{5280, ejabberd_http,    [http_poll, web_admin]}

por esto otro:

{5280, ejabberd_http,    [http_poll]},
{5281, ejabberd_http,    [web_admin, tls, {certfile, "./ssl.pem"}]}

Y a partir de ahora accede a la web con https://servidor:5281/admin/.

SciTE para Windows

En efecto, hasta que no vi la

Enviado por badlop el 16 October, 2004 - 07:37.

En efecto, hasta que no vi la administración via web de Jive 1 no entendí lo interesante que es esta forma de administración.

En Ejabberd al principio se implementó la administración via Discovery y módulo para Tkabber, pero esas funcionalidades no se van a seguir desarrollando.

Hace un par de días volvió a salir el tema, y quedó claro que a partir de ahora el camino a seguir es la administración via web. En la version de desarrollo (que no está en el CVS aun) incluso se puede editar la vCard, subscribir a nodos pub/sub y ver la traza de mensajes (no solo logs del servidor).

Vamos, que lleva camino de superar a la administración via web de Jive, que por cierto es lo único destacable que tiene ese servidor. Y en cualquier caso está muy por delante en este aspecto respecto a los otros servidores libres, que ni tienen administración via web ni Discovery siquiera.
