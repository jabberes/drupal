Jabber/XMPP ya tiene RFC

Contenido archivado

El contenido de la web se encuentra archivado y no se podrá crear nuevo contenido. Más información.

Enviado por badlop el 5 October, 2004 - 10:31.

Han sido dos años y medio de intenso trabajo de la Jabber Software Foundation y la comunidad Jabber.

XMPP-CORE, el protocolo de intercambio de mensajes entre dos entidades y XMPP-IM, la extensión para mensajería instantánea, que son las bases de Jabber ya tienen números RFC asignados:

  • RFC 3920 XMPP-CORE
  • RFC 3921 XMPP-IM
  • RFC 3922 XMPP-CPIM mapping
  • RFC 3923 XMPP end-to-end encryption

Solo por curiosidad, estos son algunos RFC conocidos con los que XMPP, y por tanto Jabber se codeará a partir de ahora:

  • 1971 RFC-0114 FTP
  • 1980 RFC-0760 IP y
    RFC-0761 TCP
  • 1981 RFC-0788 SMTP
  • 1988 RFC-1081 POP3. Su autor, Marshall T. Rose, es miembro de la Jabber Software Foundation, ha colaborado en el proceso de XMPP en la IETF y colabora en el cliente Tkabber y servidor Ejabberd.
  • 1992 RFC-1341 MIME
  • 1993 RFC-1459 IRC
  • 1996 RFC-1945 HTTP/1.0

Como alguien dice en Slashdot: 'este es un dia histórico para Jabber, para la mensajería instantánea y para Internet'.

Por supuesto, un estándar no es nada sin implementaciones. Y Jabber/XMPP tiene muchas implementaciones, muchas de ellas libres.
Algunos productos con soporte completo o casi completo de XMPP y XMPP-IM:

  • servidores libres
    • Jabberd2 en C
    • Ejabberd en Erlang
    • OpenIM en Java
    • Jive Messenger en Java también
  • clientes libres, por ejemplo:
    • Exodus, en Delphi para Windows
    • Psi, en C y Qt para Linux, Windows y MacOSX
    • Tkabber, en Tcl/Tk para Linux, Windows, MacOSX y otros
    • Pandion, en Javascript para Windows
    • Gaim, multiprotocolo, GTK para Linux y Windows
    • Kopete, multiprotocolo, Qt para Linux
  • componentes para el servidor para realizar las más diversas funciones
  • librerias libres tanto para cliente como para servidor en C, C++, C#, Java, Perl, Python, PHP, Ruby, Tcl y Delphi.

¿Qué protocolo de mensajería instantánea puede ofrecer todo esto? Solo Jabber.

¡Bienvenido/a al sitio de la comunidad de Jabber!

El sitio que estás viendo está en su mayor parte en construcción. El resultado final va a ser un sitio de Jabber orientado al usuario final. Va a contener información sobre lo que es Jabber, cómo empezar, qué son los clientes y servidores, etc., etc.

Tenemos un montón de objetivos para este sitio:

  • Debe ser realmente fácil de usar, especialmente la parte de «primeros pasos».
  • No debemos abrumar al usuario final hablándole acerca del XML, las redes distribuidas y exóticos clientes.
  • Va a ser totalmente multilingüe.
  • Debe ser el primer recurso para cada usuario (potencial) de Jabber que pueda haber.

El estado actual del trabajo está disponible en:
http://jcs.jabberstudio.org/
Usamos un sala de charla para las discusiones de los colaboradores:
jcs@conference.jabber.org
Y un wiki para la información estática:

http://www.jabber.org/wiki/index.php/JCSDocs
