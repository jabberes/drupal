El 'Jabber Glossary' original está disponible en inglés en www.jabber.org

Este 'Glosario Jabber' es una traducción directa de aquel. Traducido y mantenido por Badlop.

Hay disponible un glosario Jabber en alemán en Amessage.info



A B C D E F G H I J K L M N Ñ O P Q R S T U V W X Y Z

A

Administrador de la Sala (Room Administrator)
Alguien que puede administrar y determinar los parámetros significativos de una sala. No obstante, el Administrador de una Sala no puede destruirla, y quizá tenga poderes limitados respecto a ciertos parámetros de la misma, al contrario que el propietario de la sala.

Administrador del Servicio de Conferencia (Conferencing Service Administrator)
El administrador del servicio de conferencia de jabber.

Agente (Agent)
Componente que relaciona un servicio ajeno a Jabber y el sistema Jabber, autenticándose en el servicio ajeno por parte de un usuario Jabber.

API (API)
API es un acrónimo de Application Programming Interface. Un API es un conjunto de rutinas usadas por una aplicación, como por ejemplo el servidor Jabber.

Apodo (Nickname)
Es el nombre escogido por un usuario Jabber con el que quiere ser conocido en las charlas. El apodo puede ser distinto del nombre de usuario. En el contexto de las conferencias, usamos el término apodo de sala.

Apodo en una Sala (Room Nickname)
El apodo de un usuario Jabber en una sala de conferencia particular. Este apodo puede ser distinto de su apodo de usuario real.

Asistente (Attendee)
Cualquier usuario Jabber presente en una sala de conferencia.

Asunto (Topic)
El propósito que define una sala de conferencia. Se diferencia del tema en que no varía tan a menudo (incluso nunca) y no está ligado a una dis
cusión concreta.

Atributo XML (XML Attribute)
Los atributos XML se usan para proporcionar más información, a veces meta-información respecto a un elemento XML. Consisten en un nombre, un signo de igualdad (=) y el valor del atributo entre comillas. Un ejemplo en el protocolo Jabber sería < message to="juliet@capulet.org" >.

Autenticación (Authentication)
La autenticación es el proceso que garantiza que una entidad Jabber es un usuario válido del sistema Jabber o una parte de dicho sistema. El servidor Jabber emplea tres tipos de autenticación: contraseñas en texto llano (no recomendado), autentización con firma usando el algoritmo SHA1.

B

Biblioteca para Clientes Jabber (Jabber Client Library)
Una Biblioteca que ayuda al programador en el desarrollo de nuevos clientes Jabber, proporcionando funciones para comunicarse con servidores Jabber, transportes y servicios.

C

Centro de Conferencia(Conference Center)
Un área definida (a menudo asociada con un dominio de Internet) que hospeda salas de conferencia.

Charla (Chat)
Comunicación "uno a uno", como contraposición a la comunicación "muchos a muchos" (i.e., conferencia).

Cliente Jabber (Jabber Client)
Aplicación que permite al usuario comunicarse con servidores Jabber y así intercambiar mensajes y presencia con otros usuarios y entidades Jabber.

Componente (Component)
Un paquete de código que tiene una relación de confianza con el núcleo del servidor Jabber y proporciona entidades Jabber con funcionalidades no presentes en el núcleo del servidor (por ejemplo un servicio, agente, o transporte).

Conferencia (Conferencing)
Comunicación "muchos a muchos" entre usuarios Jabber, como contraposición a charla, que es una comunicación "uno a uno". Hay varios tipos de conferencia, entre otros conferencia de texto, de sonido, y de video.

Contacto (Contact)
Cualquier usuario Jabber al cual se está subscrito (normalmente es un miembro de un Grupo Personal o de un Grupo Compartido).

D

Directorio de usuarios Jabber (Jabber User Directory)
Un componente de código fuente abierto que permite la posibilidad de establecer una base de datos de usuarios de un servidor concreto utilizable por Jabber.

Discusión (Discussion)
Un flujo de comunicación "muchos a muchos" dentro de una sala de conferencia. Una discusión puede tener un tema, pero no es imprescindible.

Disponibilidad (Availability)
Indicador que si un usuario o entidad Jabber están disponibles para comunicarse. Ver Tipos de Disponibilidad.

DNS (DNS)
Domain Name Service (Servicio de Nombres de Dominio) es un sistema para traducir direcciones IP a direcciones FQDN, más legibles por el humano.

DTD (DTD)
Definición de Tipo de Documento (Document Type Definition), usado en XML para definir la estructura de los documentos o los datos.

E

Elemento XML (XML Element)
Los elementos XML son los bloques básicos para construir un documento XML o flujo de datos. Hay tres elementos básicos en el protocolo XML de Jabber: 'message', 'presence' y 'iq'.

Entidad Jabber (Jabber Entity)
Cada ítem independiente en Jabber que tiene un identificador Jabber. Las entidades Jabber incluyen usuarios Jabber, servidores Jabber y componentes Jabber como transportes y servicios.

Espectador(Audience Member)
Un asistente sin voz.

Estado (Status)
Ver Tipo de Disponibilidad.

F

Fichero Plano (Flat File)
Una forma de almacenar información, por ejemplo las listas de contactos, en un fichero de sistema. Lo opuesto es almacenar la información en una base de datos.

FQDN (FQDN)
Ver Fully Qualified Domain Name.

G

Grupo Compartido (Shared Group)
Una lista de usuarios Jabber y otras entidades Jabber a las que un usuario está subscrito automáticamente, normalmente a causa de su pertenencia a una parte de una organización. Contrasta con un Grupo Personal.

Grupo de charla (Groupchat)
Ver Conferencia.

Grupo personal (Personal Group)
Una lista de usuarios Jabber y otras entidades Jabber a las que se ha subscrito un usuario por su propia iniciativa. Esto contrasta con los Grupos Compartidos.

H

Handle (Handle)
Ver Apodo de Sala.

Historial de Mensajes (Message Log)
La historia de las conversaciones de un usuario Jabber con otro usuario Jabber, por lo general accesible como un archivo de HTML.

Histórico de la Discusión (Discussion Log)
Información histórica relacionada con los mensajes y cambios de presencia en una sala de conferencia o discusiones específicas dentro de una sala.

Histórico de la Sala (Room Log)
Información histórica relacionada con el uso de una sala de conferencia.

Histórico del Servicio de Conferencia (Conferencing Service Log)
Información relacionada con el uso y rendimiento generales del componente que proporciona el servicio de conferencia en el servidor.

HTML (HTML)
Lenguaje de Marcas Hiper Texto (Hyper Text Markup Language) es un subconjunto del SGML. El HTML se usa para definir la estructura y diseño de los documentos en la World Wide Web.

HTTP (HTTP)
El Protocolo de Transferencia de Hiper Texto (Hyper Text Transfer Protocol) es el modo más común de recuperar documentos de la World Wide Web. HTTP funciona a través de conexiones por TCP, normalmente al puerto 80 de los servidores web.

I

Identificador Jabber (Jabber Identifier)
Los Identificadores Jabber, abreviado 'JID', se usan externa e internamente para expresar información de pertenencia o ruta. Un Identificador Jabber está formado por dominio, nodo y recurso con el siguiente formato: usuario@anfitrión/recurso o (nodo@)dominio(/recurso).

Identificador Uniforme de Recurso (Uniform Resource Identifier)
Identificador Uniforme de Recurso. Una cadena de carácteres usada para identificar por tipo y localización un recurso (por ejemplo un fichero) desde cualquier lugar de Internet. El término 'URL' (Uniform Resource Locator, Localizador Uniforme de Recurso) es un tipo específico de URI.

IETF (IETF)
Ver Internet Engineering Task Force.

IMPP (IMPP)
El grupo de trabajo del Protocolo de Mensajería Instantánea y Presencia (Instant Messaging and Presence Protocol) de la IETF. Este grupo de trabajo fue encargado para desarrollar un estandar común para mensajería instantánea y presencia sobre Internet.

Info/Query (Info/Query)
En el protocolo Jabber, Info/Query es una estructura de datos usada para obtener información sobre una entidad Jabber o para asignar información a una entidad.

Internet Engineering Task Force (Internet Engineering Task Force)
Esta organización desarrolla protocolos para la arquitectura de Internet, principalmente a través de grupos de trabajo dedicados a areas particulares de interés.

IQ (IQ)
Ver Info/Query.

J

Jabber (Jabber)
Jabber es un sistema basado en XML para el intercambio en cuasi tiempo real de mensajes y presencia.

Jabber Open Source License (Jabber Open Source License)
JOSL es un acrónimo de Jabber Open Source License (Licencia de Código Fuente Abierto Jabber). El JOSL es una versión modificada de la Mozilla Public License (MPL, Licencia Pública de Mozilla) y ha sido aprobada por el OSI.

Jabberd (Jabberd)
Jabberd, abreviatura de Demonio Jabber (Jabber Daemon), es el proceso principal en el cual corre un servidor (o un Transporte o Servicio específico). Funciona como el centro de distribución para el servidor y maneja la lógica de encaminamiento de paquetes XML a varios componentes dentro del servidor, y eventualmente a los clientes Jabber.

JabberIM (JabberIM)
JabberIM es un cliente Jabber para Windows desarrollado por Jabber.com.

JID(JID)
Ver Identificador Jabber.

JIM (JIM)
Ver JabberIM.

JOSL(JOSL)
Ver Jabber Open Source License.

JPolld (JPolld)
El JPolld es el componente escalable que maneja los sockets (zócalos). Maneja todas las conexiones de socket del cliente y las multiplexa de regreso al servidor por una sola conexión del socket. Para proporcionar escalabilidad, las instancias JPolld pueden ser instaladas en máquinas físicas diferentes de la máquina que hospeda el Jabberd y el Manejador de Sesiones Jabber principales.

JSM (JSM)
Ver Manejador de Sesiones Jabber.

JUD (JUD)
Ver Directorio de usuarios Jabber.

K

L

LDAP
(LDAP)
Lightweight Directory Access Protocol (Protocolo Ligero de Acceso al Directorio).


Lenguaje de Marcado Extensible
(Extensible Markup Language)
El Lenguaje de Marcado Extensible (Extensible Markup Language) o XML es un formato universal para documentos y datos estructurados. El protocolo Jabber está basado en XML y XML Namespaces.


Lista de Contactos
(Roster)
Lista de usuarios Jabber y otras entidades Jabber a los que un usuario Jabber está subscrito. Esta lista es a menudo una combinación de Grupos Personales y Grupos Compartidos.


Lista negra
(Blacklist)
Lista de usuarios o dominios de Internet que tienen prohibido el acceso a salas de conferencia o el centro de conferencia.


M

Manejador de sesiones Jabber
(Jabber Session Manager)
Se encarga de todos los aspectos relacionados con la sesión del cliente en un servidor, como es el registro, autenticación, envío de mensajes y de presencia y la actualización de la lista de contactos del usuario.


Moderador
(Moderator)
La persona que dirige las discusiones y realiza ciertas funciones de administración limitadas relacionadas con una discusión de una sala de conferencia. El Moderador tiene menos privilegios que el Administrador de la Sala, en relación con la sala misma.


MIO
(MIO)
Entrada/Salida Administrada (Managed Input/Output), el API en Jabber que maneja los sockets entre Jabberd y la mayoría de componentes del servidor.


N

Namespace
(Namespace)
Ver XML Namespace.


Nombre de dominio completamente cualificado
(Fully Qualified Domain Name)
Un nombre de dominio completamente cualificado (fully qualified domain name) consiste en un nombre de máquina y de dominio, incluyendo el dominio superior. Por ejemplo, www.jabber.com es un nombre de dominio completamente cualificado, donde 'www' es la máquina, 'jabber' es el dominio de segundo nivel y '.com' es el dominio superior. El FQDN se usa para localizar una máquina en una red. Un servidor debe tener un FQDN.


Nombre de Usuario
(Username)
El nombre de cuenta de un usuario Jabber en un servidor. El nombre de usuario es la primera parte de un identificador Jabber válido. El nombre de usuario puede ser distinto del apodo del usuario.


Ñ

O

OOB
(OOB)
Fuera de Banda (Out of Band), una forma de enviar información (por ejemplo ficheros) externamente a las comunicaciones típicas Jabber. Las comunicaciones Fuera de Banda se usan normalmente para aplicaciones que consuman mucho ancho de banda, como por ejemplo transferencia de ficheros y videoconferencia.


OSI
(OSI)
La Iniciativa de Software Abierto (Open Source Initiative), una organización que determina qué licencias de software son compatibles con los principios del Software Abierto.


P

Participante
(Participant)
Es un asistente con voz.


Presencia
(Presence)
La presencia indica si una entidad Jabber está disponible para la comunicación. Incluye disponibilidad básica (o sea, conectado o desconectado) y también indicadores de estado que determinan tipos de disponibilidad.


Presencia en una Sala
(Room Presence)
La presencia de un miembro en el contexto de una sala. La presencia en una sala puede ser distinta a la presencia de ese miembro fuera de esa sala.


Prioridad
(Priority)
La prioridad es un dato definido por el usuario que determina la importancia de cada recurso que un usuario Jabber está usando para acceder al servidor. El servidor enviará mensajes e información de presencia de cada presencia al recurso con mayor prioridad. Esto permite al usuario conectarse al servidor desde diferentes lugares o dispositivos, pero recibir mensajes en solo uno de esos puntos de acceso.


Propietario de una Sala
(Room Owner)
Alguien que tiene control sobre la existencia y los parámetros fundamentales de una sala.


Protocolo
(Protocol)
Un protocolo es un conjunto formal de instrucciones que los desarrolladores deben seguir en la creación de aplicaciones, en este caso aplicaciones Jabber.


Protocolo Nativo
(Native Protocol)
Un protocolo nativo es el formato en el que ocurren las comunicaciones con un servicio particular. Por ejemplo, el Transporte ICQ se comunica con ICQ traduciendo el protocolo XML nativo de Jabber en el protocolo nativo de ICQ y viceversa.


Puerta de enlace
(Gateway)
Un componente para la parte del servidor que proporciona a los usuarios Jabber un acceso directo a servicios de mensajeria distintos de Jabber. Una puerta de enlace puede o no involucrar un mapeo tal que el servicio no Jabber pueda acceder a todo Jabber. Ejemplos de puertas de enlace son los agentes y los transportes.


Q

R

Recurso
(Resource)
El recurso de un identificador Jabber puede proporcionar información adicional respecto a cómo un usuario está accediendo al servidor. Un Recurso puede informar del lugar desde el que el usuario se está conectando (por ejemplo casa o trabajo), el tipo de dispositivo que está usando para ello (por ejemplo el portátil, o el notebook) o el cliente Jabber que está usando (por ejemplo Gabber o Psi).


Recurso Primario
(Primary Resource)
Es el recurso con mayor prioridad con el que un usuario Jabber está conectado al servidor. Es el recurso al que el servidor enviará todas las comunicaciones dirigidas a ese usuario.


RSS
(RSS)
Formato XML (inicialmente desarrollado por Netscape) para compartir titulares de noticias y resúmenes de artículos. Es un acrónimo de "RDF Site Summary" (anteriormente para "Rich Site Summary").


S

Sala
(Room)
Una forma corta de llamar a la Sala de Conferencia.


Sala de Conferencia
(Conference Room)
El escenario para una comunicación "muchos a muchos", a menudo percibido como un lugar o habitación. En Jabber, una Sala de Conferencia es un identificador Jabber (por ejemplo jdev@conference.jabber.org) al cual están subscritos los usuarios Jabber durante su participación en la discusión.


Sala no moderada
(Un-Moderated Room)
Es una sala en la que todos los participantes tienen voz (es decir, son espectadores) y en la que no hay moderador que filtre mensajes ni controle la discusión.


Sala Moderada
(Moderated Room)
Es una sala de conferencia donde no todos los participantes tienen voz (es decir, son participantes), y donde el moderador filtra mensajes y administra la discusión.


Secure Sockets Layer
(Secure Sockets Layer)
Un estándar TCP/IP que permite cifrado a nivel de comunicación de sockets.


Servicio
(Service)
Intermediario entre el núcleo de un servidor y cualquier sistema externo (por ejemplo una base de datos) en el que las comunicaciones no se realizan a través de Internet u otra red. No hay traducción entre el protocolo XML de Jabber y otro protocolo no Jabber cuando se usa un servicio. Contrasta con agente y transporte.


Servicio de Conferencia
(Conferencing Service)
El servicio ejecutado en el servidor que permite la comunicación "muchos a muchos".


Servidor Jabber
(Jabber Server)
El servidor es el programa que conecta y comunica a otros clientes y servidores Jabber. Todas las comunicaciones para un cliente se dirigen a su servidor, y de éste al cliente. Todas las preferencias y datos del cliente se almacenan en su servidor. La versión comercial del servidor se llama Servidor Jabber Comercial, mientras que la versión de código fuente abierto se llama Servidor Jabber Abierto.


Servidor Jabber Abierto
(Jabber Open Server)
El servidor de código fuente abierto y gratuito desarrollado por Jabber.org.


Servidor Jabber Comercial
(Jabber Commercial Server)
La versión comercial del servidor desarrollada por Jabber.com.


Servidores Distribuidos
(Distributed Servers)
Los Servidores Distribuidos son un sistema de servidores enlazados entre sí pero independientes, cada uno ejecutando una instancia del software. Cada servidor puede realizar todas las funciones del software sin depender del resto de servidores.


SGML (Standard Generalized Markup Language)
(SGML)
El abuelo de los lenguajes "markup". XML y HTML son subconjuntos de SGML.


SMS (Short Message Service)
(SMS)
Protocolo de mensajería instantánea para enviar mensajes cortos entre dispositivos portátiles, como teléfonos portátiles.


SMTP (Simple Mail Transfer Protocol)
(SMTP)
Estándar para correo electrónico, usado por la mayoría de los servidores de correo.


Software Abierto
(Open Source)
Software Abierto (Open Source) es un movimiento en círculos de programadores caracterizado por la creación de código 'abierto' que puede ser modificado o distribuido sin pagar una licencia. Los miembros del movimiento participan voluntariamente en el desarrollo del código y normalmente publican el nuevo código en Internet para uso común. Ver también OSI.


Sonda
(Probe)
Es un tipo específico de paquete de presencia cuyo objetivo es descubrir la disponibilidad de otra entidad.


SSL
(SSL)
Ver Secure Sockets Layer.


Subscripción
(Subscription)
Una subscripción es una petición para recibir acualizaciones de presencia de un usuario Jabber u otra entidad Jabber. La entidad receptora puede rechazar la subscripción, de forma que se impide a la parte solicitante ver actualizaciones de presencia. Si la entidad receptora aprueba la subscripción, la parte solicitante verá las actualizaciones en la presencia.


T

TCP
(TCP)
Transmission Control Protocol, un estándar de Internet para crear y manejar conexiones fiables y orientadas a flujo de datos por Internet. Jabber usa TCP para las conexiones entre el cliente Jabber y el servidor, así como entre distintos servidores Jabber.


Tema
(Subject)
Una etiqueta temporal para una discusión en una sala de conferencia.


Tipos de Disponibilidad
(Availability Type)
Indicador del estado específico o categoría de disponibilidad para la comunicación. Tradiconalmente en Jabber ha habido varios tipos de disponibilidad: "Conectado", "Ausente", "No disponible", "No molestar", y "dispuesto a charlar". El Tipo de Disponibilidad a veces recibe el nombre 'Estado'.


Transporte
(Transport)
Es un agente que proporciona acceso a un sistema de mensajería instantánea distinto a Jabber.

Transporte ICQ (ICQ Transport)
El transporte ICQ proporciona un puente entre Jabber e ICQ, un servicio de mensajería instantánea originalmente desarrollado por Mirabillis y ahora ofrecido por America Online. Ver también Transporte.

Transporte IRC (IRC Transport)
El Transporte de IRC proporciona un puente entre Jabber y el IRC (Internet Relay Chat), un sistema de mensajería instantánea abierto, principalmente usado para conferencias en modo texto. Ver también Transporte.

Transporte MSN (MSN Transport)
El Transporte MSN proporciona un puente entre Jabber y el MSN Messenger Service (MSN), un sistema de mensajería instantánea ofrecido por Microsoft Corporation. Ver también Transporte.

Transporte Yahoo (Yahoo Transport)
El Transporte de Yahoo proporciona un puente entre Jabber y Yahoo! Messenger, un servicio de mensajería instantánea ofrecido por Yahoo!. Ver también Transporte.

U

URI (URI)
Ver Identificador Uniforme de Recurso.

URL (URL)
Ver Identificador Uniforme de Recurso.

Usuario Jabber (Jabber User)
Cualquier entidad Jabber que usa el sistema Jabber con la intención de mensajería instantánea.

V

vCard (vCard)
Las tarjetas virtuales, o vCards, contienen información relativa a un usuario o servicio, de forma similar a una tarjeta de trabajo. Con ellas se automatiza el intercambio de información personal y son ampliamente usadas en gran variedad de aplicaciones de internet y telefonía.

Voz (Voice)
Es el derecho a enviar mensajes a todos los asistentes de una sala de conferencia.


W

WAP (WAP)
Protocolo para aplicaciones sin hilos (Wireless Application Protocol), proporciona a los dispositivos portátiles una forma estandarizada de acceder a información en Internet.

X

XDB (XDB)
XBD es un acrónimo de Base de datos XML (XML Data Base). Es un componente del servidor que proporciona una interfaz a cualquier tipo de fuente de datos usada para almacenar y recuperar datos de la parte del servidor.

XHTML Básico (XHTML Basic)
XHTML Básico es un subconjunto de XHTML 1.1 diseñado para clientes web tales como teléfonos móviles, PDAs, 'pagers', y 'set top box' de televisión.

XML (XML)
Ver Lenguaje de Marcado Extensible (Extensible Markup Language).

XMPP (eXtensible Messaging and Presence Protocol)
Es el protocolo usado por Jabber, también el usado por otras aplicaciones como Google Talk y LiveJournal Talk por lo que son interoperables. El protocolo está estandarizado por el IETF si bien las extensiones (XEP, XMPP Estandard Proposal, antes conocidas como JEP) extensiones a este estándar son definidas por la XMPP Software Foundation.

XML Namespace (XML Namespace)
Un XML namespace proporciona un método sencillo para calificar nombres de elementos y atributos usados en documentos XML, asociándolos con namespace identificados por una referencia URI.

Y
